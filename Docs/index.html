HTML Structure - top

This theme is structured in the latest HTML5 spec and the key areas of each page a defined below.

Page <head>

  1. Page title
  2. Meta tags for SEO
  3. Google fonts
  4. Style sheet
  5. Modernizer

Page <body>

  1. Header (logo & navigation)
  2. Content (page specific layouts)
  3. Footer (footer columns and copyright)
  4. Scripts (jQuery, plugins, custom scripts, Google analytics)

Updating the logo

The logo is setup to work with retina displays and requires you to create two separate logo's. One at the original size (eg. 100x50px) and one at double the size (eg. 200x100px). The double size is called logo@2x.png with the "@2x" representing double scale.

Update files:
images/logo.png
images/logo@2x.png
css/style.css (lines 80-81)

Components - Columns

This theme uses a flexible 1140px grid as to scale to all screen dimensions

Basic structure - First create a wrapper div with a class of "clearfix", then inside this div create some more div's with the appropriate classes for your desired width and make sure the last div to the right has the class "last" to stop it overflowing.

View the columns template for all possible layouts

                        <div class="clearfix">
                            <div class="one-half"></div>
                            <div class="one-half last"></div>
                        </div>

Components - Icons

This theme uses font icons which are perfect for retina displays and are simple to customise with CSS. The basic structure for a font icon is below contained in a simple link. The class="icon" will activate the custom font and the data-icon="" will define what icon is used based off a letter in the alphabet. The full listing of icons and their respective letter is in the elements.html template

                        <a href="#">
                            <span class="icon" data-icon="s"></span>
                            Link with icon 
                        </a>

CSS Files and Structure - top

The CSS directory is broken into two areas, style.css and the fonts folder. All styling is done in the style.css file and the fonts folder is to house any custom fonts that you wish to host with the website.

style.css

This is broken into key sections from top to bottom

  1. Normalize.css
  2. Custom fonts - hosted
  3. Base styles (including headings)
  4. Link styles
  5. General helper classes (including page width)
  6. Custom grid
  7. Layout and element styles (navigation, content, sidebar, widgets)
  8. Plugins
  9. Media queries (tablet & mobile)
  10. Print styles

Custom fonts

This site uses a mixture of custom fonts, we have font icons that are hosted with the site in the fonts directory and referenced in the top of styles.css

This site also uses Google Fonts

Google fonts are referenced in the head of each page and can be easily changed:

  1. Visit http://www.google.com/fonts/ and select a font
  2. Update the head with the supplied link code from Google
  3. Update the references in style.css to the new font-family name.
    eg. font-family: 'Droid Sans', sans-serif;

JavaScript - top

The javascript is split between two main files, plugins.js and script.js and these are referenced in the the page footer right before the closing body tag.

plugins.js

This is simply a merged version of all the plugins needed in the theme and is done to speed up the loading time. The original files are located in the js/libs directory for reference.

scripts.js

This is where all the interactions are located. The layout is using the javascript namespace format to make sure we can load only the needed scripts when a certain class/ID is found on a page.

 

Lightbox

To activate a lightbox simply add either of the below options to a link

  • class="lightbox" - for a single lightbox or
  • rel="lightbox" to create a gallery lightbox
                        <a href="image.jpg" class="lightbox">
                            Open lightbox
                        </a>

More info can be found here: http://fancybox.net/

Slider

This theme uses the responsive Flexslider, all documentation and customisation can be found here: http://www.woothemes.com/flexslider/

For a basic slider just create a div with the class="flexslider", then add and an unordered list with the class="slides" inside with the different slides in each list item. See the blog listing template for a simple example.

To change the animation to slide or to change the timing between each slide simply un-comment the option(s) in script.js flexslider function

Twitter feed

NOTE! Twitter is changing its API and this feature may cease to work in the future. If a work around becomes available I will be sure to include it in a future update.

To add your own twitter feed simply add your twitter username in the plugin options in scripts.js (line 112)

More info and options can be found here: http://tweet.seaofclouds.com/

Map

The map is powered by the jQuery gMap plugin.

The map requires two extra scripts (google maps api and gMaps plugin) so these are added in the footer above the reference to plugins.js

To configure the map markers and position, update the plugin options in scripts.js (line 235). To get the lat and long for a map marker use this site: http://itouchmap.com/latlong.html.

More info at http://www.smashinglabs.pl/gmap

Videos

To make embedded videos responsive you only need to wrap the iframe code in a div with the class="video" and your done.

                        <div class="video">
                            <iframe src="youtube.com" ...
                        </div>

API Usage - top

Flickr - Photo stream

The site only uses one manual API call and this is for the flickr photo stream. To configure the stream you only need to update scripts.js (line 203) and add the desired user ID to the URL.

To get a users ID use http://idgettr.com/

Update this part of the URL - 53358796@N05 - with the desired user ID
